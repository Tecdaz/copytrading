# Tooling Specification

## Purpose

Define the toolchain contract for the `copytrading` project so every subsequent change can be developed, linted, type-checked, and tested with zero additional setup. Verified by a smoke test and five CLI commands that MUST all exit 0 on a clean clone.

## Requirements

### Requirement: Project Skeleton

The repo SHALL contain a `src/`-layout package named `copytrading`, generated by `uv init --package copytrading`. Entry point: `src/copytrading/__init__.py`. Metadata: `pyproject.toml` at the repo root with `requires-python = ">=3.13,<3.14"`.

#### Scenario: Skeleton exists on a clean clone

- GIVEN a fresh clone, Python 3.13, uv 0.10+
- WHEN the user runs `uv sync`
- THEN the command exits 0, `src/copytrading/__init__.py` exists, and `[project] name = "copytrading"` is declared

### Requirement: Runtime and Dev Dependencies

`pyproject.toml` MUST declare runtime deps `py-clob-client`, `google-api-python-client`, `google-auth`, `python-dotenv` and dev deps `pytest`, `pytest-cov`, `ruff`, `mypy`. The lockfile `uv.lock` MUST be committed.

#### Scenario: Lockfile is reproducible

- GIVEN the committed `uv.lock`
- WHEN a second clone runs `uv sync`
- THEN resolved versions are byte-identical to the first clone

### Requirement: Ruff Lint and Format Configuration

`pyproject.toml` MUST contain `[tool.ruff]` with `line-length = 100`, `target-version = "py313"`, and rule selection `E F I N UP B SIM RET` applied to `src/` and `tests/`. A `[tool.ruff.format]` block MUST be present.

#### Scenario: Ruff passes on a clean tree

- GIVEN the configured `pyproject.toml`
- WHEN the user runs `uv run ruff check .` and `uv run ruff format --check .`
- THEN both exit 0

### Requirement: Mypy Strict Type Checking

`pyproject.toml` MUST contain `[tool.mypy]` with `strict = true` and `python_version = "3.13"`. A `[[tool.mypy.overrides]]` block MUST relax `disallow_untyped_defs` for `tests/` only. A second override MUST set `ignore_missing_imports = true` for `py_clob_client`, `googleapiclient`, and `google.oauth2`.

#### Scenario: Strict mypy passes

- GIVEN the configured `pyproject.toml` and the smoke test
- WHEN the user runs `uv run mypy src tests`
- THEN the command exits 0

### Requirement: Pytest Test Runner

`pyproject.toml` MUST contain `[tool.pytest.ini_options]` with `testpaths = ["tests"]` and `addopts = "-ra -q"`. The `tests/` directory SHALL exist at the repo root.

#### Scenario: Pytest discovers the smoke test

- GIVEN `tests/test_smoke.py`
- WHEN the user runs `uv run pytest`
- THEN the command exits 0 and reports at least one passed test

### Requirement: Package Version Export

`src/copytrading/__init__.py` MUST define `__version__ = "0.1.0"` as a string, MUST include the docstring `"""copytrading — Polymarket copy-trading bot."""`, and MUST NOT contain placeholder `hello.py` content.

#### Scenario: Importing the package exposes the version

- GIVEN the package installed via `uv sync`
- WHEN a process executes `from copytrading import __version__`
- THEN `__version__` is a `str` equal to `"0.1.0"`

### Requirement: Smoke Test Contract

`tests/test_smoke.py` MUST contain `test_version_is_string` that imports `__version__` from `copytrading` and asserts it is a `str` equal to `"0.1.0"`.

#### Scenario: Smoke test fails on version drift

- GIVEN a developer edits `__version__` to `"0.2.0"` without updating the smoke test
- WHEN the user runs `uv run pytest`
- THEN the test fails citing expected vs actual version

### Requirement: Ignore and Example Environment Files

The repo root MUST contain `.gitignore` covering `.venv/`, `__pycache__/`, `*.db`, `.env`, `logs/`, `.mypy_cache/`, `.ruff_cache/`, `.pytest_cache/`, `*.egg-info/`, `dist/`, `build/`. The repo root MUST also contain `.env.example` with placeholders for `POLY_API_KEY`, `POLY_SECRET`, `POLY_PASSPHRASE`, `POLY_PRIVATE_KEY`, `GOOGLE_SHEETS_CREDENTIALS_PATH`, `GOOGLE_SHEET_ID`.

#### Scenario: Secrets never get committed

- GIVEN a developer copies `.env.example` to `.env` and fills credentials
- WHEN the developer runs `git status`
- THEN `.env` is reported as ignored

### Requirement: README Onboarding

The repo root MUST contain `README.md` with a one-paragraph description and the canonical commands `uv sync`, `uv run pytest`, `uv run ruff check .`, `uv run ruff format .`, `uv run mypy src tests`.

#### Scenario: New developer bootstraps from the README

- GIVEN a fresh clone and a new developer reading `README.md`
- WHEN the developer runs the five listed commands in order
- THEN every command exits 0
